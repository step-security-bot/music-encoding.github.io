Tercer paso: conocer los principales elementos hijo del elemento raíz <mei>: <meiHead> y <music>.

¡Buen trabajo! Ahora que hemos declarado un espacio de nombres dentro de nuestro elemento raíz, todos los demás elementos anidados en este pueden resolverse de acuerdo con el espacio de nombres MEI mediante el procesamiento automático.

Como ya se ha mencionado, se puede imaginar que un documento XML es como un árbol que se ramifica desde un elemento raíz hasta los elementos hijo anidados. Un archivo codificado con MEI contiene al menos dos estructuras dentro del elemento raíz <mei>:

  • <meiHead>: elemento que contiene un conjunto de elementos que proporcionan metadatos relacionados con la codificación MEI, incluyendo información bibliográfica, declaraciones de responsabilidad, normas de codificación, o procedencia.
  • <music>: elemento que contiene un conjunto de elementos que proporcionan la codificación del contenido musical. Entre otras cosas, puede contener elementos estructurales para describir la forma de una pieza musical (como secciones, movimientos, o grupos de documentos), o elementos de notación (como notas, silencios, articulaciones, o dinámicas).
