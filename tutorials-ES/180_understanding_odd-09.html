Macros constituyen la última parte de ODD que se necesita para comprender plenamente las especificaciones de MEI, y son claramente el aspecto menos importante. Los macros son marcadores de posición para elementos que pueden añadirse al <content> de un elemento de una sola vez. El macro más utilizado es probablemente macro.struc-unstrucContent, que se define como sigue: 

<macroSpec ident="macro.struc-unstrucContent" module="MEI.shared" type="pe">
        <desc>Ofrece la posibilidad de elegir entre contenido estructurado y no estructurado/mixto.</desc>
        <content>
            <rng:choice>
                <rng:group>
                    <rng:zeroOrMore>
                        <rng:ref name="model.headLike" />
                    </rng:zeroOrMore>
                    <rng:zeroOrMore>
                        <rng:ref name="model.pLike" />
                    </rng:zeroOrMore>
                </rng:group>
                <rng:zeroOrMore>
                    <rng:choice>
                        <rng:text/>
                        <rng:ref name="model.textPhraseLike.limited" />
                    </rng:choice>
                </rng:zeroOrMore>
            </rng:choice>
        </content>
    </macroSpec>

Los macros vuelven a utilizar el elemento <macroSpec>, pero esta vez con un @type con valor "pe". Los nombres, que de nuevo se almacenan en @ident, llevan como prefijo un "macro.", y por supuesto volvemos a tener un <desc>. La principal diferencia con las clases de modelo es que los macro no sustituyen sólo a los elementos, sino también a reglas más complejas sobre combinaciones de elementos. Aquí, macro.struc-unstrucContent proporciona un buen ejemplo. Obliga al usuario a decidir entre un modelo de codificación estructurado (que utiliza un elemento <head> dedicado, seguido de una serie de párrafos), o un modelo no estructurado (en el que el contenido textual se inserta directamente y puede marcarse con una variedad de elementos). Se puede utilizar dentro de elementos como <watermark>, y es útil para evitar tener que redefinir la misma combinación de elementos en numerosos lugares. 

<elementSpec ident="watermark" module="MEI.header">
    <desc>Contiene una descripción de una marca de agua o dispositivo similar.</desc>
    <classes>
        <!-- memberships omitido por brevedad -->
    </classes>
    <content>
        <rng:ref name="macro.struc-unstrucContent" />
    </content>
    <!-- observaciones omitidas por razones de brevedad -->
</elementSpec>

En esencia, el <rng:ref> dentro del <elementoEspec> es simplemente reemplazado por los elementos hijos del <macroSpec>'s <content>, de modo que es posible combinar un macro con otros contenidos. Por lo tanto, macros sirven como bloques de especificaciones predefinidos y reutilizables.

Como mencioné antes, macros no son el aspecto más importante de la ODD desde la perspectiva del MEI, por lo que ya deberías haber alcanzado todo lo necesario. Eso significa que has terminado con este último paso - vamos a continuar para algunos comentarios finales...
