Personalización de ODD

Este es un tutorial avanzado que presenta cómo adaptar MEI a las necesidades específicas de un proyecto. Asume una sólida comprensión de las estructuras ODD internas de MEI, como se describe en el tutorial Comprendiendo ODD tutorial. Si no estás familiarizado con elementos <elementSpec> , clases modelo y clases atributo, probablemente sea mejor volver al tutorial primero.

Empecemos con nuestro tutorial actual. El propósito de las personalizaciones es restringir el esquema MEI schema a sólo aquellas partes del marco que son relevantes para una tarea o proyecto determinado. También es posible extender MEI utilizando los mismos mecanismos descritos aquí, pero esto se deja para otro tutorial más. Por ahora, la atención se centra en el subconjunto de MEI, ya que los archivos validados contra dicha versión personalizada de MEI también validará usando la personalización MEI All, que se asemeja al marco marco MEI. Esto garantiza un nivel sustancial de compatibilidad con las herramientas y la documentación existentes. En general, las ampliaciones de MEI deben hacerse sólo cuando sea absolutamente necesario, y en tales situaciones suele ser una buena idea consultar a la Comunidad MEI para verificar que no existe otra solución ya existe.

En este tutorial, aprenderemos primero a utilizar el MEI ProfileDrafter, que proporciona una interfaz gráfica de usuario para generar personalizaciones restrictivas básicas. Forma parte del MEI Garage, y aunque es algo limitado, definitivamente ayuda a comprender mejor los conceptos de las personalizaciones MEI. A continuación, examinaremos la salida de MEI ProfileDrafter y veremos cómo generar a partir de ella un archivo RelaxNG con fines de validación. Por último, presentaremos algunas opciones de personalización más avanzadas, que están fuera del alcance de MEI ProfileDrafter. Es posible que en algún momento desees utilizar una aplicación de edición XML , pero utilizaremos servicios web para todo el procesamiento, por lo que no importa qué editor elijas - cualquiera está bien. ¡Comencemos!
