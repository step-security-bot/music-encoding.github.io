El MEI normal permite elementos de documento: <mei>, <meiHead>, <music>, y <meiCorpus>. Esto significa que estos elementos pueden servir como elementos más externos en un archivo MEI. Sin embargo, un proyecto puede idear una estructura de datos que requiera otros elementos iniciales o adicionales. Por ejemplo, un proyecto puede decidir mantener todas las anotaciones editoriales como archivos separados, cada uno con su propio <annot> elemento. La personalización oficial mei-all_anyStart permite utilizar cualquier elemento del espacio de nombres MEI como elemento inicial, pero esto es sobre todo con fines didácticos y no es recomendable para un uso práctico. En su lugar, ODD ofrece un mecanismo sencillo para definir los elementos de inicio permitidos, usando el elemento inicial <schemaSpec>. 

<schemaSpec ident="mei" 
    ns="http://www.music-encoding.org/ns/mei" 
    prefix="mei_" 
    start="mei meiHead meiCorpus music">

Como puedes ver, el atributo @start se utiliza como una lista separada por espacios de todos los elementos de inicio. Si el proyecto mencionado decidiera permitir <mei> and <annot>, pero desautorizar todos los demás elementos de inicio, esto podría modificarse así: 

<schemaSpec ident="mei" 
    ns="http://www.music-encoding.org/ns/mei" 
    prefix="mei_" 
    start="mei annot">

Como puedes ver, ajustar los elementos de inicio es bastante fácil. Terminemos con lo que hemos aprendido sobre la personalización de ODD.
